# Neovim Configuration

Personal Neovim minimal configuration for version 0.12+
Uses native Neovim features wherever possible — no external plugin managers.

## Structure

```
~/.config/nvim/
├── init.lua                        # Entry point — loads all modules in order
├── nvim-pack-lock.json             # Auto-generated lockfile for vim.pack
└── lua/
    ├── config/
    │   ├── options.lua             # Editor options (signcolumn, etc.)
    │   ├── keymaps.lua             # Global keymaps (completion, snippets, etc.)
    │   └── colors.lua              # Color palette (generated by Matugen)
    └── plugins/
        ├── lsp.lua                 # LSP setup: Mason + nvim-lspconfig + completion
        ├── tokyonight.lua          # Colorscheme (overridden with Matugen colors)
        ├── telescope.lua           # Fuzzy finder
        ├── gitsigns.lua            # Git blame and diff signs in the gutter
        ├── autopairs.lua           # Auto-close brackets and quotes
        ├── cord.lua                # Discord Rich Presence
        ├── tmux-navigator.lua      # Seamless navigation between Neovim and tmux panes
        └── languages/
            └── vala.lua            # Syntax highlighting for Vala
```

## Plugin Manager

Plugins are managed with **`vim.pack`**, the native package manager built into Neovim 0.12+.
No external plugin manager (lazy.nvim, packer, etc.) is required.

Plugins are declared with:
```lua
vim.pack.add({ { src = "https://github.com/author/plugin" } })
```

Neovim downloads and installs them automatically on first launch.
The `nvim-pack-lock.json` file pins the exact commit of each plugin.

## LSP

LSP servers are installed via **Mason** (`:Mason` to open the UI).
All installed servers are automatically enabled at startup via `mason-lspconfig`.

Completion is powered by the **native Neovim LSP completion** (no nvim-cmp).
It only triggers when the character before the cursor is non-whitespace, to avoid
popups on empty lines or after spaces.

## Colors

The colorscheme is **Tokyonight** with colors overridden by the **Matugen** palette,
generated from the current wallpaper. The palette is written to `lua/config/colors.lua`
by Matugen's template system and loaded automatically by `plugins/tokyonight.lua`.
